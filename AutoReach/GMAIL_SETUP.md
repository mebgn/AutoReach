# Gmail Setup Guide for `.env`

This guide explains how to get Gmail SMTP info and fill your `.env` file for the bulk email sender.

---

## 1) Gmail SMTP values

Use these fixed values:

- `SMTP_HOST=smtp.gmail.com`
- `SMTP_PORT=587`
- `USE_TLS=true`

Use your Gmail address for:

- `SMTP_USERNAME=your_gmail@gmail.com`
- `SENDER_EMAIL=your_gmail@gmail.com`

For password, use a **Google App Password** (not your normal Gmail password):

- `SMTP_PASSWORD=your_16_character_app_password`

---

## 2) How to get Google App Password

1. Open: https://myaccount.google.com/
2. Go to **Security**
3. Enable **2-Step Verification**
4. Go to **App passwords**
5. Select **Mail** (or choose Custom name)
6. Copy the generated 16-character password
7. Put it in `SMTP_PASSWORD` inside your `.env`

If **App passwords** is missing, usually:
- 2-Step Verification is not enabled, or
- your account policy/admin blocks it.

---

## 3) Example `.env`

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your_email@gmail.com
SMTP_PASSWORD=abcd efgh ijkl mnop
SENDER_EMAIL=your_email@gmail.com
USE_TLS=true

DEFAULT_SUBJECT=Internship Application - Mohamed Bougaydane
DEFAULT_COMPANY=your company
DEFAULT_NAME=Hiring Team

SEND_DELAY_SECONDS=3
MAX_EMAILS_PER_RUN=
LOG_FILE=sent_log.csv
```

> If needed, you can remove spaces from app password.

---

## 4) Run command (easy mode)

Once `.env`, `emails.csv`, and `resume.pdf` are ready, run:

```bash
python3 main.py
```

This command sends emails to all valid recipients in `emails.csv`.

---

## 5) Why `LOG_FILE=sent_log.csv`?

This is the output log generated by the script. It records:

- recipient email
- subject and company used
- status (`SENT`, `DRY_RUN`, `SKIPPED`, `FAILED`)
- failure reason (if any)
- timestamp

It helps you track outreach and avoid duplicate sends.
